Sarjy 🎙️

A bilingual voice assistant that remembers you across sessions — and books your dinner, hands-free. Pre-read for the 5-minute presentation.

sarjy-production-3bc0.up.railway.app — try it now: tap the orb once and talk (English or عامية مصرية). Append ?debug for live latency instrumentation.

What it does

  • Hands-free voice: tap once — voice-activity detection hears when you finish, answers out loud, and keeps listening. Tap while it speaks to interrupt.
  • Memory across sessions: tell it your name or favorite color, close the browser, come back tomorrow — it remembers. Every fact is visible and deletable in the Memories drawer.
  • Multi-step restaurant booking (the deep dive): collects details across turns, validates them, searches real restaurants (Foursquare + Geoapify geocoding), tolerates interruptions and corrections, requires explicit confirmation, and resumes unfinished bookings in later sessions.
  • Bilingual + multimodal: full Arabic support through the entire workflow (بكرة → tomorrow's date, نعم confirms); attach a photo — "book for the number of people in this picture" fills the party size by counting them.

Why Foursquare (the external API)

A booking workflow without real data is a form-filler. Foursquare makes Sarjy a recommender — actual venues at actual malls the user knows. Geoapify supplies geocoding because Foursquare's own geocoder fails on Cairo districts and Arabic names — each API doing what it's best at. A no-hallucination contract is enforced by tests: restaurant names enter the conversation only as validated tool results; failures are reported honestly.

The deep dive: the LLM proposes, the FSM disposes

The workflow is a deterministic finite-state machine — COLLECTING → PRESENTING → CONFIRMING → COMPLETED — the LLM only extracts slots and speaks. Four guarantees no prompt can provide:

  • Per-state tool legality — confirming outside CONFIRMING is structurally impossible
  • Per-field validation — "party of 250 tomorrow at 8" keeps the date, rejects the size with a reason
  • A confirmation double-guard — booking requires the user's literal yes (English or Arabic); neither model hallucination nor prompt injection can book
  • Persistence every turn — unfinished bookings resume across sessions

The entire machine runs its 30+ tests with zero network and zero LLM.

Architecture

Modular monolith: FastAPI, one Docker image on Railway, SQLite behind repositories on a persistent volume. One turn: Whisper STT → gpt-4o-mini with tools → streamed TTS (audio plays before synthesis finishes). Moderation screens every input in parallel with the LLM — safety at zero latency cost, verdict checked before any tool executes. Memory recall is prompt injection, not retrieval: at dozens of facts per user, RAG could only add a miss mode (deliberate, documented). A background extraction sweep makes fact-capture deterministic even when the chat model just answers.

Engineering hygiene

  • 100 tests, zero network, in CI on every push + a smoke gate run against production
  • Rate limiting (proven live at request 21), moderation guardrails, injection tests, sensitive-data guard on memory
  • Request-id-correlated logs, per-turn metrics, token/cost tracking
  • Graceful degradation at every hop — no failure reaches the user as a stack trace

Trade-offs made deliberately

  • Request/response over full-duplex streaming — demo reliability over the last 1.5 s; next step is token-streaming into sentence-chunked TTS (~2 s TTFA)
  • SQLite over Postgres — single-writer, zero-ops, volume-persisted (survival verified across redeploys); repository seam makes the swap a connection string
  • No RAG — full prompt injection = 100% recall at this scale
  • Simulated reservation — no open booking API exists here; the FSM's confirmation gate is where a partner API would plug in

Found-in-testing stories (ask me about these)

  • Arabic users hit an infinite confirmation loop — the FSM's affirmation guard was doing its job, but only spoke English. One regex later, نعم books dinner.
  • The model silently skipped saving a 4-fact Arabic introduction → the deterministic background extraction sweep was born.
  • Foursquare's geocoder returns an empty boundary for "New Cairo" while two Sizzlers sit 8 km from its center — hence Geoapify coordinates.
